OTBI Pipeline Skeleton — pkg_otbi_soap
=======================================

This package provides SOAP infrastructure for executing OTBI (Oracle
Transactional BI) logical SQL queries via the BI Web Services API.
Individual load_* procedures are generated by the Fusion Data Pipeline
Generator and added to this package.

Prerequisites
-------------
1. Oracle APEX workspace on your database
2. A Fusion Cloud service account with OTBI query access
3. Your Fusion instance base URL (e.g., https://abcd-test.fa.ocs.oraclecloud.com)

Setup
-----
1. No constants to configure — credentials and instance URL are passed
   as parameters to each procedure.

2. Compile the spec first, then the body:
   @pkg_otbi_soap.sql
   @pkg_otbi_soap.plb

3. Use the Pipeline Generator (OTBI mode) to generate load_* procedures.
   Copy each generated procedure into the package body before the
   final END statement, and add the spec declaration.

What's Included
---------------
- parse_session_id() — extracts session ID from SOAP logon response
- get_session()      — authenticates via SOAP logon (private)
- execute_sql_query()— runs logical SQL via executeSQLQuery (private)
- run_otbi_query()   — full pipeline: logon -> query -> XML parse (public)

Notes
-----
- Credentials (username/password) are passed as parameters, not stored
  in the package. Each load_* procedure accepts p_username, p_password,
  and p_instance_url.
- The substr(response, 40) call strips a BOM/leading characters from
  the SOAP response before XML parsing. This is a known quirk of the
  Oracle BI SOAP service.
